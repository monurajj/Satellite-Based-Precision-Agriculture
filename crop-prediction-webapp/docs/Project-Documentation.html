1. Project Overview

The Crop Yield Predictor is a full-stack web application that allows small farmers to estimate crop production (in tons) based on field conditions. It uses the trained machine learning model from the main Satellite-Based Precision Agriculture project (XGBoost/Random Forest on satellite and weather data).

Key Features

  • Prediction Form: Farmers enter crop type, land area, soil type, rainfall, temperature, and solar radiation.
  • Results Page: Displays predicted yield per hectare and total yield in tons.
  • Dashboard: Shows previous predictions in a table and analytics charts (bar chart, pie chart).
  • Real ML Model: Predictions come from the trained model (best_model.joblib), not dummy formulas.

3. End-to-End Workflow

When a farmer submits a prediction request, this is what happens:

1. User fills form (crop type, land area, soil type, rainfall, temp, solar) and clicks "Get Prediction".
2. Frontend sends POST request to /api/predict (proxied to backend:4000).
3. Backend validates input, spawns predict_ml.py, passes JSON via stdin.
4. Python script loads training stats, maps farmer inputs to 24 model features, loads best_model.joblib, runs model.predict(), returns JSON via stdout.
5. Backend receives result, stores prediction in memory, returns JSON to frontend.
6. Frontend navigates to Results page with prediction data, displays yield clearly.
Data flow: Farmer inputs → Backend validation → Python feature mapping → ML model inference → Backend storage → Frontend display

How Frontend Links to Backend

The frontend uses a proxy configured in vite.config.js:

proxy: {
  '/api': {
    target: 'http://localhost:4000',
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/api/, ''),
  },
}

When the frontend calls fetch('/api/predict', ...), Vite forwards the request to http://localhost:4000/predict. This avoids CORS issues during development.

5. Backend (Node.js + Express)

Structure

backend/
├── server.js      # Express app, routes, Python subprocess call
├── prediction.js  # In-memory storage: addPrediction(), getHistory()
├── predict_ml.py  # Python ML inference script
└── package.json

How the Backend Works

  1. POST /predict: Receives JSON body. Validates required fields (cropType, landArea, soilType). Calls callMLPredict(input), which spawns Python, writes JSON to stdin, reads JSON from stdout, returns prediction. Stores result in memory and responds.
  2. GET /history: Returns array of stored predictions (newest first).
  3. GET /health: Simple health check.

Python Subprocess Integration

// Backend spawns: python predict_ml.py
// Sends: JSON.stringify(input) via stdin
// Receives: JSON.parse(stdout) with predictedYield, totalYield, unit, message

The backend uses the project's venv/bin/python so the Python script has access to the same environment (numpy, pandas, scikit-learn, joblib).

6. ML Model Integration

Model Source

The model is trained by the main project's main.py pipeline and saved at:

experiments/results/best_model.joblib

It uses features from satellite data (NDVI, spectral bands), weather (precip, temp, solar, GDD), and soil (pH, OC, clay).

Farmer Inputs → Model Features

Since farmers don't provide satellite data, predict_ml.py maps their inputs to the model's expected feature space:

Farmer InputModel Features
Soil typesoil_pH, soil_OC, soil_clay (from SOIL_PARAMS mapping)
Rainfall (mm)precip_sum (scaled to training range ~150–300)
Temperature (°C)temp_mean, gdd_sum (GDD base 5°C)
Solar (W/m²)solar_sum (scaled to training range)
—ndvi_peak, cumulative_ndvi, etc. (derived from rain/temp)
—B2–B12, pca_0–2 (training median; no satellite data)

Feature Order (24 features)

ndvi_peak, season_length, cumulative_ndvi, ndvi_grain_fill_slope,
precip_sum, temp_mean, solar_sum, gdd_sum, soil_pH, soil_OC, soil_clay,
B2, B3, B4, B8, B11, B12, pca_0, pca_1, pca_2,
ndvi_peak_x_precip_sum, ndvi_peak_x_gdd_sum, ndvi_peak_x_soil_OC, gdd_sum_x_precip_sum

8. Setup & Run Instructions

Prerequisites

  • Node.js 18+
  • Python 3.9+ (with project venv)
  • Trained model: run python main.py in main project first

Step 1: Train the ML Model (once)

cd Satellite-Based Precision Agriculture
source venv/bin/activate  # or: venv\Scripts\activate on Windows
python main.py

This creates experiments/results/best_model.joblib and data/features.csv.

Step 2: Install Backend Dependencies

cd crop-prediction-webapp/backend
npm install

Step 3: Install Frontend Dependencies

cd crop-prediction-webapp/frontend
npm install

Step 4: Start Backend

cd crop-prediction-webapp/backend
npm start

Backend runs at http://localhost:4000.

Step 5: Start Frontend (new terminal)

cd crop-prediction-webapp/frontend
npm run dev

Frontend runs at http://localhost:3000. Open in browser.

Quick Test

curl -X POST http://localhost:4000/predict \
  -H "Content-Type: application/json" \
  -d '{"cropType":"Wheat","landArea":5,"soilType":"Loam","rainfall":500,"temperature":20,"solarRad":150}'
